# 결제 신뢰성 실험실
### 동기식 결제 승인 시스템을 비동기 처리, 모니터링, 멱등성, 복구 전략까지 확장하며 신뢰성을 학습한 프로젝트 💵

---

## 결제 시스템 요구사항

1. 외부 PG의 결제 상태와 내부 서버의 결제 상태는 최종적으로 일관되어야 합니다.(정합성)
2. 외부 PG 결제는 중복 승인되면 안되며, 장애 시 복구가 가능해야합니다.(안정성)
3. 외부 PG 결제 요청 시 클라이언트의 결제 정보는 바뀔 수 없습니다.(무결성)
4. 결제 이벤트는 되도록 빠르게 발견 가능해야하며, 추적 가능해야 합니다.(운영성)
5. 리틀의 법칙 사용하여 계산(내일 추가)

## v1. 동기식 결제 시스템

동기식 결제 시스템은 3개의 큰 흐름으로 이루어져 있습니다.

1. 결제 요청 API : 결제 창을 띄우기 전 DB에 결제 정보를 저장하는 API
2. 결제 창 : 클라이언트는 결제 창 호출/표시 후 결제 수단과 인증을 진행한다.
3. 결제 승인 API : 서버가 최종적으로 토스 결제 승인 API를 호출해, 돈을 실제 결제한다.
